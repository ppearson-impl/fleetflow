[✓] SUGGESTION TYPES (6 TOTAL)

    1. REORDER_STOPS
       - Detects inefficient stop sequencing (>30% above optimal)
       - Uses nearest-neighbor algorithm
       - Complexity: EASY
       - Typical savings: 15-25% distance reduction

    2. CONSOLIDATE_STOPS
       - Identifies geographically clustered stops (<0.5 km)
       - Suggests consolidation strategies
       - Complexity: MEDIUM
       - Typical savings: 5-10% per cluster

    3. CHANGE_VEHICLE
       - Recommends vehicle downsizing when underutilized
       - Triggers: Weight util <50% AND stops <=8
       - Complexity: MEDIUM
       - Typical savings: 15-20% fuel costs

    4. SPLIT_ROUTE
       - Suggests route splitting for large routes
       - Triggers: Stops >12 OR (stops >8 AND weight >85%)
       - Complexity: HARD
       - Typical savings: 10-15% distance reduction

    5. CONSOLIDATE_STOPS
       - Groups geographically close deliveries
       - Reduces handling time and travel
       - Complexity: MEDIUM

    6. TIME_WINDOW_OPTIMIZATION
       - Identifies tight early morning time windows
       - Recommends customer negotiation
       - Complexity: MEDIUM
       - Typical savings: 10-30 minutes

================================================================================
ALGORITHM DETAILS
================================================================================

HAVERSINE DISTANCE:
- Uses great-circle distance between GPS coordinates
- Accurate for all distances
- Returns result in kilometers
- Formula: d = R x 2 x atan2(sqrt(a), sqrt(1-a))

NEAREST-NEIGHBOR OPTIMIZATION:
- Greedy algorithm for route optimization
- Selects nearest unvisited stop at each step
- Time complexity: O(n2)
- Practical heuristic (not guaranteed optimal)

CLUSTERING:
- Identifies stops within 0.5 km radius
- Transitive closure for cluster membership
- Supports up to 2 clusters per analysis
- O(n2) complexity
